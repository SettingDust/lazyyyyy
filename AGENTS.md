## Before Writing Code
- Read all relevant files first. Do not edit blindly.
- Fully understand the requirements before starting to code.

## When Writing Code
- Test after writing. Do not leave untested code.
- Fix errors before continuing. Do not skip failures.
- Prefer editing over rewriting entire files.
- The simplest viable solution. Do not overdesign.

## Before Declaring Completion
- Run the code once more to confirm it works.
- Do not declare completion without passing tests.

## Output
- No flattering introductions or meaningless conclusions.
- No dashes, smart quotes, or Unicode. ASCII only.
- Keep it concise. If unsure, state so. Do not guess.

## Override Rules
User instructions always take priority over this file.

## Project Context Recording
- Record reusable project information into the memory system as context reference for subsequent tasks.
- Content to record includes:
  - Project structure and tech stack
  - Common classes, utility methods, and design patterns
  - Code style and naming conventions (such as indentation, bracket style, naming standards, etc.)
  - Project-specific development practices and best practices
  - Solutions to common problems
- Prioritize using recorded context information in subsequent tasks.
